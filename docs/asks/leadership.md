# Modulo 5 - Liderazgo Tecnico

## 1) 5 prioridades en primeras 2 semanas
1. Frenar incidentes criticos: triage diario y estabilizacion de servicios clave.
2. Definir estandares minimos obligatorios (DoD, code review, testing baseline).
3. Reducir riesgo tecnico en componentes con mayor impacto de negocio.
4. Alinear roadmap de 3 semanas con alcance realista y hitos semanales.
5. Plan de mentoring para los 2 juniors en tareas acotadas y revisadas.

## 2) Organizacion del equipo
- Dividir en dos frentes: estabilizacion (incidentes/deuda alta) y feature prioritaria.
- Asignar lider tecnico por frente.
- Pair programming selectivo senior-junior.
- Daily enfocada en bloqueos y riesgos, no en status largo.

## 3) Metricas de desempeno
- Lead time y cycle time por historia.
- Change failure rate y MTTR.
- Cobertura de pruebas y defectos en produccion por release.
- Throughput semanal y cumplimiento de compromisos.

## 4) Practicas tecnicas obligatorias
- Pull request con minimo un revisor.
- Pruebas automatizadas para flujos criticos.
- Checklist de seguridad y observabilidad por cambio.
- Definition of Done comun para todo el equipo.
- Alertas y runbooks para servicios criticos.

## 5) Gestionar presion de negocio sin comprometer calidad
- Negociar alcance, no calidad: entregar MVP seguro con fases.
- Exponer riesgos tecnicos con impacto cuantificado.
- Reservar capacidad fija para deuda/incidentes (ej. 30%).
- Publicar tablero de avance y riesgos para transparencia ejecutiva.
